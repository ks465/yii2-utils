KHanModel Class ¶

Documentation Edition: 1.1-971112 Class Version: 0.4.6-971027

All of the standard globally required methods and properties are set here for simplicity. Some of these methods are gathered in the main model, and some are in the accompanied khans\utils\models\queries\KHanQuery The most important items are:

  1. Listing errors in validating a model in text format
  2. Disabling Delete method and replace it with status-changing method.
  3. Save method forcefully validates the attributes. If validation succeeds, a flash message is added to session. If validation fails, the model errors are added to flash message.
  4. Adding BlameAble and TimeStamp behaviors to automatically record the creation and update time and user.
  5. The status field shows how to filter (or when to show) the item:
    • KHanModel::STATUS_DELETED shows the record is totally out of reach of every user and method except for the Super Admin.
    • KHanModel::STATUS_PENDING shows the record is out of order. No record in other models can choose the value of this record for creating or updating values. But whenever there is need to show data of other records which depend in this one, data is available.
    • KHanModel::STATUS_ACTIVE shows data is completely available for updating, using in other data sets, etc.
  6. Methods to find the creator, last updater, and time of each.
    • If there is more than one table holding users --like pGrad-- change:
      • khans\utils\models\KHanModel::getResponsibleUser to be able to see all the users in the above methods,
      • khans\utils\helpers\migrations\KHanMigration::$user_models to create all the tables and models,
  7. isActive Shows if the status of the record is equal to KHanModel::STATUS_ACTIVE
  8. isVisible Shows if the status of the record is NOT equal to KHanModel::STATUS_DELETED
  9. getLastFlow shows the updated time
  10. Get list of changes in the record through the history data

For details see khans\utils\models\KHanModel and khans\utils\models\queries\KHanQuery

A sqlite database is ready in the demos directory, which contains all models needed for the demos. In order to activate it, load db component as follows:

'db' => [
    'class' => 'yii\db\Connection',
    'dsn' => 'sqlite:@app/db/database.db',
    'charset' => 'utf8',
    // Schema cache options (for production environment)
    //'enableSchemaCache' => true,
    //'schemaCacheDuration' => 60,
    //'schemaCache' => 'cache',
],
