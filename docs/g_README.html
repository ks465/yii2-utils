Guide ¶

This is document for KHanS/yii2-utils package.

Documentation Edition: 2.2-971122

Notice: You need both supportEmail and adminEmail to be set.

Documentation ¶

Guides, notes, and anything regarding good style programming and documenting:

  1. Document Tips and Tricks Useful data for using documents, demos and also how to update documents and run tests.
  2. Setup for Persian Settings and tweaks for activating and tuning pages for RTL Persian language.
  3. Complete Config A complete config.php file for utilizing full power
  4. Workflow Setup for workflow, workflow manager, and workflow components.

Components ¶

All the classes in the component directory of the package:

  1. Array Helper contains more methods for transforming and reshaping arrays.
  2. File Helper contains methods to read and write data in different formats.
  3. Jalali and JalaliX contain Jalali date object creation and manipulation.
  4. Math Helper contains methods to do some mathematical routines easier.
  5. SQL Formatter --a third party class-- formats a given SQL query for better visibility.
  6. String Helper contains all the utilities for formatting, correcting and shaping strings for saving or showing.
  7. View Helper contains all the utilities required for simpler and better view of objects in output.

Controllers ¶

  1. KHan Console Common methods for console applications.
  2. KHan Rest Common Method for REST applications.
  3. KHan Web Common methods for web applications.
  4. System Tables Methods for updating System Tables.

Columns ¶

Columns are classes extending \kartik\grid\ActionColumn instead of \yii\grid\ActionColumn. They are specialized versions for Kartik Gridview.

  1. Action Column contains ActionColumn for GridViews.
  2. Boolean Column contains BooleanColumn for GridViews.
  3. Data Column contains DataColumn for GridViews.
  4. Enum Column contains EnumColumn for GridViews.
  5. Jalali Column contains a Jalali date column for GridViews.
  6. Progress Column contains a workflow progress status for GridView.
  7. Radio Column contains RadioColumn for GridViews to select row and work with bulk action.
  8. Related Column

Models ¶

  1. Base Model contains basic skeleton for all models.
  2. Base user contains basic skeleton for all user models.
  3. Identity contains basic skeleton for all user identity models.

Others ¶

These are miscellaneous classes for configuring the package or installation.

  1. App Builder contains methods useful for creating models, controllers, CRUD packages.
  2. System table Builder contains methods to create or update system tables from definitions in the kernel tables
  3. Generators contains multiple generator templates for use besides GII.
  4. VarDump contains methods to make life easier and fun for the admins in debugging the code objects.
  5. SqlFormatter contains methods to make life easier and fun for the admins in debugging the SQL queries.
  6. Settings contains all the settings and constant values.
  7. Migrations contains all the migrations required for rising the package up and running.
  8. Overlay Menu contains description of an overlay menu, which covers all the browser page. This can be used inside NavBar menu or as a button on the page.
  9. Workflow Manager contains all the requirements for builidng, using and managing workflows
  10. Wizard Flow
  11. Tests
  12. Tools Module contains a module for some tools
    • History of changes
    • History of login attempts
    • EAV Attributes

Sanitizing User Input ¶

Add following code to application options in order to filter or otherwise sanitize every user input: 

'on beforeRequest' => [
    '\khans\utils\components\StringHelper',
    'screenInput',
],

RBAC Setup ¶

Add following code to component section of application options in order to setup AuthManager tables:

  • Setting identityClass is not necessary, KHanUser has set this value to \khans\utils\models\KHanIdentity
  • class should be set to one of the child classes of \khans\utils\models\KHanUser and NOT itself. This class should implement public static function tableName()
'user' => [
    'class' => '\app\models\UserTable',
    //'identityClass'   => '\khans\utils\models\KHanIdentity',
    'enableAutoLogin' => true,
    'identityCookie' => [
        'name' => '_identity_name_',
        'httpOnly' => true,
        'path' => '_path_to_application_',
    ],
],
'authManager' => [
    'class' => 'yii\rbac\DbManager',
    'ruleTable' => 'sys_auth_rule',
    'itemTable' => 'sys_auth_item',
    'itemChildTable' => 'sys_auth_item_child',
    'assignmentTable' => 'sys_auth_assignment',
],

Run ./yii migrate/up --migrationPath=@khan/helpers/migrations to build required tables with the given names.
