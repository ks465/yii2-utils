Guide ¶

This is document for KHanS/yii2-utils package.

Documentation Edition: 1.0-970803

Documentation ¶

Guides, notes, and anything regarding good style programming and documenting:

  1. Document Tips and Tricks
  2. Setup for Persian

Actions ¶

  1. CSV Grid View shows data saved in a CSV file as a \KHanS\Utils\widgets\GridView.

Components ¶

All the classes in the component directory of the package:

  1. Array Helper contains more methods for transforming and reshaping arrays.
  2. File Helper contains methods to read and write data in different formats.
  3. Jalali and JalaliX contain Jalali date object creation and manipulation.
  4. Math Helper contains methods to do some mathematical routines easier.
  5. SQL Formatter --a third party class-- formats a given SQL query for better visibility.
  6. String Helper contains all the utilities for formatting, correcting and shaping strings for saving or showing.
  7. View Helper contains all the utilities required for simpler and better view of objects in output.

Columns ¶

Columns are classes extending \kartik\grid\ActionColumn instead of \yii\grid\ActionColumn. They are specialized versions for Kartik Gridview.

  1. Action Column contains ActionColumn for GridViews.
  2. Boolean Column contains BooleanColumn for GridViews.
  3. Data Column contains DataColumn for GridViews.
  4. Enum Column contains EnumColumn for GridViews.
  5. Radio Column contains RadioColumn for GridViews to select row and work with bulk action.

Models ¶

  1. Base Model contains basic skeleton for all models.
  2. Base user contains basic skeleton for all user models.
  3. CSV DataProvider

  1. Date Picker

Others ¶

These are miscellaneous classes for configuring the package or installation.

  1. VarDump contains methods to make life easier and fun for the admins in debugging the code objects.
  2. SqlFormatter contains methods to make life easier and fun for the admins in debugging the SQL queries.
  3. Settings contains all the settings and constant values.
  4. Migrations contains all the migrations required for rising the package up and running.
  5. Overlay Menu contains description of an overlay menu, which covers all the browser page. This can be used inside NavBar menu or as a button on the page.

StartUp ¶

General requirements in application options:

    'language' => 'fa-IR',
    'timeZone' => 'Asia/Tehran',
     'aliases' => [
        '@khan' => '@vendor/khans465/yii2-utils',
     ],
     'modules' => [
         'gridview' => [
             'class' => '\kartik\grid\Module',
         ],
     ],
     'components' => [
        'formatter' => [
            'class' => 'yii\i18n\Formatter',
            'nullDisplay' => '_NULL_value_to_display_',
            'booleanFormat' => ['_FALSE_', '_TRUE_'],
            'locale' => 'fa_IR@calendar=persian',
            'decimalSeparator' => '-',
            'thousandSeparator' => '_',
            'numberFormatterOptions' => [ //These two require PHP intl extension
                NumberFormatter::MIN_FRACTION_DIGITS => 0,
                NumberFormatter::MAX_FRACTION_DIGITS => 2,
            ]
        ],
        'session' => [
            'name' => '_staff_session',
            'savePath' => __DIR__ . '/../runtime/sessions',
            //'class' => 'yii\web\DbSession',
            //'sessionTable' => 'sys_session_staff',
            'cookieParams' => [
                'httpOnly' => true,
            ],
        ],
        'i18n' => [
            'translations' => [
                'app' => [
                    'class' => 'yii\i18n\PhpMessageSource',
                    'basePath' => '@app/messages',
                ],
                'kvgrid' => [
                    'class' => 'yii\i18n\PhpMessageSource',
                    'basePath' => '@app/messages',
                ],
            ],
        ],
        ...
     ],

Add following code to console options in order to setup migrations table:

'controllerMap' => [
    'migrate' => [
        'class' => 'yii\console\controllers\MigrateController',
        'migrationTable' => 'sys_migration',
    ],
],

Sanitizing User Input ¶

Add following code to application options in order to filter or otherwise sanitize every user input: 

'on beforeRequest' => [
    '\KHanS\Utils\components\StringHelper',
    'screenInput',
],

RBAC Setup ¶

Add following code to component section of application options in order to setup AuthManager tables:

  • Setting identityClass is not necessary, KHanUser has set this value to \KHanS\Utils\models\KHanIdentity
  • class should be set to one of the child classes of \KHanS\Utils\models\KHanUser and NOT itself. This class should implement public static function tableName()
'user' => [
    'class' => '\app\models\UserTable',
    //'identityClass'   => '\KHanS\Utils\models\KHanIdentity',
    'enableAutoLogin' => true,
    'identityCookie' => [
        'name' => '_identity_name_',
        'httpOnly' => true,
        'path' => '_path_to_application_',
    ],
],
'authManager' => [
    'class' => 'yii\rbac\DbManager',
    'ruleTable' => 'sys_auth_rule',
    'itemTable' => 'sys_auth_item',
    'itemChildTable' => 'sys_auth_item_child',
    'assignmentTable' => 'sys_auth_assignment',
],

Run ./yii migrate/up --migrationPath=@yii/rbac to build required tables with the given names.
