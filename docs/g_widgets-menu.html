OverlayMenu Class ¶

  1. subsection
  2. OverlayMenuFiller
  3. OverlayMenuAsset
  4. CSV structure

This is a simple widget class to render the menu. It consists of:

  • OverlayMenuFiller
  • OverlayMenuAsset
  • views/menu
echo OverlayMenu::widget([
    'title'      => 'General Menu',
    'label'      => 'This Is the Output',
    'tag'        => 'button',
    'csvFileUrl' => '@khan/tests/demos/sample-menu.csv',
    'options'    => ['class' => 'btn btn-danger'],
    'tabs'       => [
        'general'    => [
            'id'    => 'general',
            'title' => 'General',
            'icon'  => 'heart',
            'admin' => false,
        ],
        'others'     => [
            'id'    => 'others',
            'title' => 'Others',
            'icon'  => 'user',
            'admin' => false,
        ],
        'management' => [
            'id'    => 'management',
            'title' => 'Manager',
            'icon'  => 'alert',
            'admin' => true,
        ],
    ],
]);

The above code renders a button on the page to activate the overlay menu. By rendering the widget, a button is placed on the page. You can render the button in the NavBar. The tabs section is optional. The \KHanS\Utils\widgets\menu\OverlayFiller creates it if omitted.

OverlayMenuFiller ¶

This is the part which builds the data array, which later is rendered in the views/menu. It is not necessary to build the filler separately. You can configure the main class.

OverlayMenuAsset ¶

This an AssetBundle class which publishes the required CSS and JS files to the web root.

CSV structure ¶

Current implementation requires the following fields to be present in the CSV file.

  1. id Primary key of the data
  2. title String to place in the link.
  3. url Target of the menu link.
  4. image Icon for each item. --NOT in use currently
  5. class Display class for texts.
  6. tab Id of the section --also column-- of the menu.
  7. order Sorting number.
